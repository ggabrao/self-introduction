API

The following table emulates the main CRUD operations over a REST API, using an In-Memory Web Api for Angular. It allow us to simulate data persistence operations by intercepting requests that would otherwise go to a remote server. That way, no backend is needed for testing HTTP requests.

You can add a new course, edit or delete, and then see the updated list.

Main Features

Route Resolvers

Were applied in some routes so that the view can only be displayed after the data from HTTP requests have been gathered. That can improve the user experience, since the view will not appear with empty values and suddenly refresh all filled up.

In-Memory Data Storage

Allow us to simulate data persistence operations for testing purpose, including auto-generating id's. It's a better option than hard-coding the data.

Component Communication

Table, Dialog and Update component are all integrated, so the data can flow among them with consistency, allowing any changes to be shared.

Angular Material

The table, the dialog and the forms were build using Angular Material components.
